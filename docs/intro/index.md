# Welcome to Secuird

Secuird replaces SSH key management with a Certificate Authority model. Your servers trust one CA. Users sign in with Google, GitHub, or Microsoft and get short-lived certificates scoped to exactly what they're allowed to access.

---

## Pick your path

### As an admin
You set up the organization, generate the CAs, invite the team, and configure who can access what.
→ [Admin setup guide](/docs/intro/how-it-works#admin-flow)

### As a user 
You accept the invitation, complete MFA enrollment if required, and register your SSH public key.
→ [User onboarding](/docs/intro/how-it-works#user-flow)

### As a developer
You use the CLI to log in once, register your key, and request certificates.
→ [Developer workflow](/docs/intro/how-it-works#developer-flow)
